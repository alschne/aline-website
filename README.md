# Site README

## Overview

This is a static website hosted on GitHub Pages. It is built with plain HTML, CSS, and JavaScript, with one addition: [Eleventy (11ty)](https://www.11ty.dev/) is used as a static site generator to power the `/articles/` section.

The rest of the site (home, about, services, contact, FAQ, etc.) is plain HTML and is not touched by Eleventy.

---

## Tech Stack

| Layer | What it is |
|---|---|
| Hosting | GitHub Pages (main branch, root domain) |
| Styles | Single stylesheet at `/css/style.css` |
| Shared UI | Header/footer loaded via `/includes.js` fetching `/header.html` and `/footer.html` |
| Blog/Articles | Eleventy SSG — templates in `/src/`, markdown in `/src/articles/` |
| Node | Required only for Eleventy — not used elsewhere |

## How the Articles Section Works

1. Markdown files in `/src/articles/` are picked up by Eleventy
2. Each file is processed through `article.njk` → `base.njk` to produce a full HTML page
3. Eleventy writes the output directly into `/articles/` in the repo root
4. The articles listing page (`/articles/`) is also generated by Eleventy from `src/articles/index.njk`
5. All generated pages share the same header, footer, and stylesheet as the rest of the site

The nav active state (highlighting "Articles" when on any article page) is handled automatically by the existing `includes.js` logic.

## Local Development

Requires Node.js. Install once with Homebrew if not already installed:

```bash
brew install node
```

Install dependencies (first time only):

```bash
npm install
```

Start local dev server with live reload:

```bash
npm run dev
```

Visit `http://localhost:8080/articles/` to preview.

Build static files for deployment:

```bash
npm run build
```

---

## Deploying

```bash
npm run build
git add .
git commit -m "your message"
git push
```

GitHub Pages serves from the main branch root. Eleventy writes directly into `/articles/` so no separate deploy step is needed.

---

## Key Conventions

- **Never manually edit anything inside `/articles/`** — it is fully overwritten by Eleventy on every build
- **Article images** live in `/images/` and are referenced by path in frontmatter
- **Nav changes** go in `/header.html` — one file, applies to the entire site including generated article pages
- **Style changes** go in `/css/style.css` — one file, applies to everything
